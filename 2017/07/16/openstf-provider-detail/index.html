OpenSTF Provider模块详解

简介

OpenSTF通过Provider模块进行设备的跟踪与管理。在Provider模块中使用adbkit中的client.trackDevices([callback])方法可获取到设备的状态，比如有新设备连接或移除。

启动 Provider

可通过 provider [serial..] 命令行启动provider子进程。
主要配置参数：
heartbeat-interval：心跳检测时间间隔。
lock-rotation：是否锁定屏幕方向。
max-port：可分配给设备的最大tcp端口号。
min-port：可分配给设备的最小tcp端口号。
mute-master：使用设备时是否静音。

设备过滤器

filterDevice方法，用于在发现新设备时检查设备连接是否正常。
以下情况为连接不正常，返回false：
1.device.id无法获取或错误。
2.非远程连接模式下通过远程方式连接的设备。
3.与启动参数中的serial不一致。
除以上情况外该方法会返回一个device对象，表示新设备。

发现新设备连接

调用adbkit.trackDevices通过回调方法获取tracker对象，并注册add事件。当发现一台新设备连接时先通过设备过滤器判断该设备是否可用，如果可用则检查设备类型正常后创建并启动device子进程。

检查设备类型

当设备类型为device或emulator则创建并启动device子进程。
